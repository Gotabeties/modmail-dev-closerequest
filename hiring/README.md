Adds a Hiring Request Menu panel with private controls to add, edit, and delete hiring requests.
The menu shows your current open count (example: `0/3`). Each user can keep up to 3 open requests at a time.
The panel message is sent as an embed.
All submissions validate Discord invite links, save to Supabase, and post an embed to a configured channel.
If `hiringconfig usepaneloutput true` is enabled, the panel embed is automatically re-sent so it stays at the bottom.
Each posted hiring embed includes the request ID/UUID in the footer.
Hiring listing embeds always use the title `New Hiring Post`.
Hiring post embeds use your bot `recipient_color`, and panel/menu embeds use `main_color`.
All hiring embeds can optionally include a configured image at the bottom.

Panel setup commands:

- `hiringconfig setpanelchannel <#channel>`
- `hiringconfig setpanelmessage <message>`
- `hiringconfig sendpanel`
- `hiringconfig setoutputchannel <#channel>`
- `hiringconfig usepaneloutput <true|false>`
- `hiring setembedtitle <title>`
- `hiringconfig setembedimage <image_url|none>`
- `hiring requestinfo <id_or_uuid>`
- `hiring deleterequest <id_or_uuid>`
- `hiring relistactive`

`hiring setembedtitle` updates the panel/menu embed title.

## Supabase Setup

Use `hiringconfig setsupabase <url> <key> [table]`.

- `url`: your project URL from Supabase Dashboard → **Settings** → **API** → **Project URL**
- `key`: a Supabase API key from Supabase Dashboard → **Settings** → **API** → **Project API keys**
	- Recommended: use the `service_role` key for server-side bot usage

### Required Table Columns

Create table `hiring_submissions` (or your custom table name) with these columns:

- `guild_id` (text)
- `guild_name` (text)
- `user_id` (text)
- `username` (text)
- `company_name` (text)
- `position` (text)
- `description` (text)
- `discord_server_link` (text)
- `submitted_at` (timestamptz)

Example SQL:

```sql
create table if not exists public.hiring_submissions (
	id bigint generated by default as identity primary key,
	guild_id text not null,
	guild_name text not null,
	user_id text not null,
	username text not null,
	company_name text not null,
	position text not null,
	description text not null,
	discord_server_link text not null,
	submitted_at timestamptz not null default now()
);
```

If you get a duplicate key error like `Key (guild_id)=... already exists`, your table was created with `guild_id` as primary key/unique.
Fix it with:

```sql
alter table public.hiring_submissions drop constraint if exists hiring_submissions_pkey;
alter table public.hiring_submissions add column if not exists id bigint generated by default as identity;
alter table public.hiring_submissions add primary key (id);
alter table public.hiring_submissions drop constraint if exists hiring_submissions_guild_id_key;
```
